# Notes: WI-024 — Fix token_usage Null in spawn_session Normal-Path Response

## Context

In `mcp/session-spawner/server.py:582–583`:
```python
if outcome_token_usage is not None:
    response["token_usage"] = outcome_token_usage
```

When token extraction fails (e.g., `output_format="text"`, or JSON parse fails), `token_usage` is absent from the response. The timeout path at line 567 correctly emits `"token_usage": null`. Observability P-3 requires explicit null rather than omission.

## Fix

Replace lines 582–583 with:
```python
response["token_usage"] = outcome_token_usage
```

When `outcome_token_usage` is `None`, Python's `json.dumps` serializes it as `null`. When it is a dict, it serializes as an object. This is a 2-line reduction.
