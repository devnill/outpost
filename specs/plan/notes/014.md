# Notes: 014 — Fix poll_remote_job Missing Timestamp Fields

## Context

Source: cycle 2 review gap analysis.

In `_poll_one()` (mcp/session-spawner/server.py around line 843–847), completed/failed job responses from the remote worker include `created_at`, `started_at`, and `completed_at` fields in the HTTP response body, but the field copy loop only copies:

```python
for field in ("output", "git_diff", "exit_code", "duration_ms", "error"):
    if field in body:
        result[field] = body[field]
```

The three timestamp fields are silently dropped.

## Fix

Extend the field copy tuple to include the timestamp fields:

```python
for field in ("output", "git_diff", "exit_code", "duration_ms", "error",
              "created_at", "started_at", "completed_at"):
    if field in body:
        result[field] = body[field]
```

The `if field in body` guard already handles the case where a field is absent (e.g., `started_at` is None for queued jobs, `completed_at` is None for running jobs — these won't be in the body or will be null).

## Verification

No new tests required — the change is additive and the guard prevents errors when fields are absent. Confirm existing tests still pass.
