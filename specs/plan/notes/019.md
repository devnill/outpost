# Notes: WI-019 — Add cancel_remote_job MCP Tool

## Context

WI-017 implemented `DELETE /jobs/{job_id}` on the remote worker for both queued and running jobs. But there's no MCP tool in session-spawner to call it. Callers must hit the worker's HTTP API directly.

## New Tool Definition

Add to `list_tools()` after the `poll_remote_job` Tool entry (~line 200):

```python
Tool(
    name="cancel_remote_job",
    description=(
        "Cancel a previously submitted remote job. "
        "Cancels jobs in 'queued' or 'running' state. "
        "Returns 'cancelled' status on success, or an error if the job cannot be cancelled "
        "(already completed, failed, or not found)."
    ),
    inputSchema={
        "type": "object",
        "properties": {
            "job_id": {
                "type": "string",
                "description": "The job ID returned by spawn_remote_session.",
            },
            "worker_name": {
                "type": "string",
                "description": (
                    "Name of the worker that owns this job. "
                    "If omitted, all configured workers are tried in config order "
                    "until the job is found."
                ),
            },
        },
        "required": ["job_id"],
    },
),
```

## Dispatch

In `call_tool()` (~line 242), add:
```python
if name == "cancel_remote_job":
    return await _handle_cancel_remote_job(arguments)
```

## Handler Implementation

Model `_handle_cancel_remote_job` closely on `_handle_poll_remote_job` (lines 817–901). Key differences:

- Use `session.delete(url)` instead of `session.get(url)`
- For 204: `return [TextContent(type="text", text=json.dumps({"job_id": job_id, "status": "cancelled", "worker_name": w["name"]}))]`
- For 409: parse body for `detail`, return `{"error": body.get("detail", f"HTTP 409"), "worker_name": w["name"]}`
- For 404: mark `{"_status": "not_found", "_worker": w["name"]}`
- For 401/403: mark `{"_status": "auth_error", ...}`
- Fan-out: try workers **sequentially** in config order (not parallel); return first definitive result (204, 409, auth error). If all 404, return not-found error.

**Important**: DELETE 204 has no response body. Do NOT call `resp.json()` on a 204 response.

## aiohttp DELETE call

```python
async with _get_http_session().delete(
    f"{w['url'].rstrip('/')}/jobs/{job_id}",
    headers={"X-API-Key": w.get("api_key", "")},
    timeout=conn_timeout,
) as resp:
    if resp.status == 204:
        # success
    elif resp.status in (401, 403):
        # auth error
    elif resp.status == 404:
        # not found
    elif resp.status == 409:
        body = await resp.json()
        # conflict
```
