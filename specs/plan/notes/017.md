# Notes: 017 — Implement Running Job Cancellation

## Context

Source: user decision (2026-03-16 interview). remote-dispatch Q-5.

Currently `cancel_job` (DELETE /jobs/{job_id}) returns HTTP 409 for running jobs. The `cancelled` state is defined but unreachable from `running`. Users have no way to abort a long-running remote job short of restarting the daemon.

## Implementation Plan

### 1. Add process field to JobRecord

```python
class JobRecord:
    def __init__(self, job_id: str, request: JobRequest):
        ...
        self.process: subprocess.Popen | None = None  # set before communicate() in _run_claude_job
```

### 2. Change _run_claude_job to use Popen

Replace `subprocess.run(...)` with `Popen` + `communicate()`. Store the Popen object on the record before calling `communicate()` so the cancel endpoint can signal it.

Since `_run_claude_job` is called via `asyncio.to_thread`, `record` is accessible from both the worker thread and the async cancel endpoint. The Popen object is set before blocking on `communicate()`.

```python
def _run_claude_job(record: JobRecord) -> tuple[str, int, str | None]:
    cmd = [...]

    try:
        proc = subprocess.Popen(
            cmd,
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE,
            text=True,
            cwd=record.working_dir,
        )
        record.process = proc  # set before communicate() so cancel can signal it

        try:
            stdout, stderr = proc.communicate(timeout=record.timeout)
        except subprocess.TimeoutExpired:
            proc.kill()
            stdout_bytes, stderr_bytes = proc.communicate()
            partial_stdout = stdout_bytes if isinstance(stdout_bytes, str) else stdout_bytes.decode("utf-8", errors="ignore")
            return partial_stdout, -1, f"Job timed out after {record.timeout}s"

        error = stderr if proc.returncode != 0 else None
        return stdout, proc.returncode, error
    finally:
        record.process = None  # clear after completion
```

Note: `record.process = None` in the finally block is a cleanup step. The cancel endpoint checks `record.status` first — if the job completes before cancel reaches it, the endpoint sees `status != "running"` and returns 409. Race condition is acceptable since job completion is the desired outcome.

### 3. Update cancel_job endpoint

```python
@app.delete("/jobs/{job_id}", status_code=204)
async def cancel_job(job_id: str):
    async with job_store_lock:
        record = job_store.get(job_id)
        if not record:
            raise HTTPException(status_code=404, detail="Job not found")

        if record.status == "queued":
            record.status = "cancelled"
            return None

        if record.status == "running":
            proc = record.process
            record.status = "cancelled"
            record.completed_at = datetime.datetime.now(
                datetime.timezone.utc
            ).isoformat(timespec="milliseconds").replace("+00:00", "Z")
            # Signal outside the lock to avoid blocking
        else:
            raise HTTPException(
                status_code=409,
                detail=f"Cannot cancel job with status '{record.status}'",
            )

    # Signal the process after releasing the lock
    if proc is not None:
        proc.terminate()
        # Give it 2s to terminate gracefully, then kill
        await asyncio.sleep(0)  # yield to event loop
        try:
            await asyncio.wait_for(
                asyncio.to_thread(proc.wait),
                timeout=2.0
            )
        except asyncio.TimeoutError:
            proc.kill()

    return None
```

### 4. Handle cancelled status in _process_job

After `_run_claude_job` returns, `_process_job` sets `record.status` to `completed` or `failed`. If the job was cancelled while running, `cancel_job` already set `record.status = "cancelled"`. The `_process_job` logic should not overwrite a `cancelled` status:

```python
async with job_store_lock:
    record.output = output
    record.exit_code = exit_code
    record.git_diff = git_diff
    record.error = error
    record.duration_ms = duration_ms
    if record.status != "cancelled":  # don't overwrite cancel
        record.completed_at = completed_at
        record.status = "failed" if exit_code != 0 else "completed"
```

## Important: Thread Safety

`record.process` is set in the worker thread (before `communicate()`) and read in the async cancel endpoint. Python's GIL provides sufficient protection for this single reference assignment. No additional locking needed beyond what's already there.
