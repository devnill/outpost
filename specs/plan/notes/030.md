# WI-030: Fix conftest sys.modules key collision between test suites

## Context

Both conftest files register their server module under `sys.modules["server"]`. When pytest collects both directories in a single session (e.g., `pytest mcp/`), the second registration silently overwrites the first. Future bare `import server` would resolve the wrong module.

Tracked as OQ-021 since cycle 6.

## Files to Modify

1. `mcp/session-spawner/conftest.py` — change registration key
2. `mcp/remote-worker/conftest.py` — change registration key
3. `mcp/session-spawner/test_server.py` — update any references to the old module key
4. `mcp/remote-worker/test_server.py` — update any references to the old module key

## Fix

### mcp/session-spawner/conftest.py

Change:
```python
sys.modules["server"] = module
```
To:
```python
sys.modules["session_spawner_server"] = module
```

### mcp/remote-worker/conftest.py

Change:
```python
sys.modules["server"] = module
```
To:
```python
sys.modules["remote_worker_server"] = module
```

### Test files

Read both test files and check for any bare `import server` or `sys.modules["server"]` references. The conftests use `importlib` to load the module and register it — the test files typically access the module via a fixture or via the conftest-registered name. Update any references that use the old string key `"server"` to use the appropriate new key (`"session_spawner_server"` or `"remote_worker_server"`).

## Verification

After the change, run `pytest mcp/` from the repository root and confirm:
- All 121 tests pass
- No `ImportError` or module resolution errors
- No test accidentally exercises the wrong server module
