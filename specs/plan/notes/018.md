# Notes: WI-018 — Propagate Role system_prompt for Remote Sessions

## Context

`_handle_spawn_remote_session` (server.py:678) applies `allowed_tools`, `permission_mode`, and `max_turns` from a resolved role but never prepends `system_prompt` to the prompt payload. This was introduced as a regression by WI-015. The local `spawn_session` path does this correctly at lines 289–292.

## Fix Location

In `_handle_spawn_remote_session`, after role resolution (lines 682–693) and before payload construction (line 752). Insert immediately after line 693:

```python
# system_prompt: prepend to the prompt (matching local spawn_session behavior at lines 289-292)
if resolved_role and resolved_role.get("system_prompt"):
    role_label = role_arg if isinstance(role_arg, str) else "custom"
    arguments = dict(arguments)  # avoid mutating the original dict
    arguments["prompt"] = f"[ROLE: {role_label}]\n{resolved_role['system_prompt']}\n\n{arguments['prompt']}"
```

Then at payload construction (line 752), `payload["prompt"] = arguments["prompt"]` picks up the injected version automatically — no further change needed.

**Alternative**: Modify the prompt field in the payload directly after construction. Either approach is correct; the above is simpler since `arguments["prompt"]` is already referenced in the payload line.

## Roles with system_prompt (from default-roles.json)

- `worker`: no system_prompt → no injection
- `reviewer`: has system_prompt ("You are a code reviewer...") → inject with label "reviewer"
- `manager`: has system_prompt → inject with label "manager"
- `proxy-human`: has system_prompt (very long) → inject with label "proxy-human"
