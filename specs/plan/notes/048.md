## `_run_claude_job` changes

Add a branch at the top of `_run_claude_job` that builds a Docker command when `_agent_image` is set:

```python
def _run_claude_job(record: JobRecord) -> ...:
    if _agent_image:
        cmd = _build_container_cmd(record)
    else:
        cmd = _build_claude_cmd(record)
    ...
```

Extract the existing command-building into `_build_claude_cmd(record)` and add `_build_container_cmd(record)`:

```python
def _build_container_cmd(record: JobRecord) -> list[str]:
    cmd = ["docker", "run", "--rm"]
    if _container_runtime:
        cmd.extend(["--runtime", _container_runtime])
    cmd.extend([
        "--name", f"job-{record.job_id}",
        "--user", "1000:1000",
        "--cap-drop", "ALL",
        "--security-opt", "no-new-privileges",
        "--memory", _container_memory,
        "--memory-swap", _container_memory,
        "--cpus", _container_cpus,
        "--pids-limit", "512",
        "-v", f"{record.working_dir}:/workspace",
        "-e", f"ANTHROPIC_API_KEY={os.environ.get('ANTHROPIC_API_KEY', '')}",
        _agent_image,
        "--print",
        "--output-format", "json",
        "--permission-mode", "dangerouslySkipPermissions",  # always inside container
        "--max-turns", str(record.max_turns),
        "--cwd", "/workspace",  # inside container, not host path
    ])
    if record.allowed_tools:
        cmd.extend(["--allowedTools", ",".join(record.allowed_tools)])
    cmd.append(record.prompt)
    return cmd
```

Note: `--cwd` inside the container is always `/workspace` (the bind mount target), not `record.working_dir`.

## container_name lifecycle

Set `record.container_name` before `Popen` is called (when in container mode) and clear it in the `finally` block:

```python
if _agent_image:
    record.container_name = f"job-{record.job_id}"
try:
    proc = subprocess.Popen(cmd, ...)
    ...
finally:
    record.process = None
    record.container_name = None  # clear after container exits
```

## Cancel path changes

In `cancel_job` (the `DELETE /jobs/{job_id}` endpoint, around line 298), add a Docker stop call when `container_name` is set. This happens after the lock is released, alongside the `proc.terminate()` call:

```python
if proc is not None:
    # Stop the container explicitly if in container mode
    if container_name:
        try:
            subprocess.run(
                ["docker", "stop", container_name],
                timeout=15,
                capture_output=True,
            )
        except (subprocess.TimeoutExpired, FileNotFoundError, OSError):
            pass
    try:
        proc.terminate()
    except (ProcessLookupError, OSError):
        pass
    ...
```

Store `container_name = record.container_name` inside the lock (same as `proc = record.process`) before releasing.

## Backward compatibility

When `_agent_image` is empty (default), the function falls back to the existing `_build_claude_cmd` path and behaves identically to before.

## ANTHROPIC_API_KEY injection note

The API key is passed via `-e ANTHROPIC_API_KEY=...` in the `docker run` command. This means:
- The key appears in the process argument list (visible to `ps aux` on the host)
- Alternative: use `--env-file` with a temp file per job, or Docker secrets
- For the initial implementation, `-e` is acceptable; document the exposure and note `--env-file` as a Tier 2 improvement
