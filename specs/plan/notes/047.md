# WI-047 Implementation Notes: Dockerfile for outpost agent container image

## File to create

`mcp/remote-worker/Dockerfile`

## Reference implementation

Base on the Anthropic devcontainer pattern from `anthropics/claude-code/.devcontainer/Dockerfile`:
- Base: `node:20-bookworm-slim` (smallest Debian-based Node.js image with full apt support)
- Install git (for `_capture_git_diff` to work inside future container-native paths, and for agent tasks that use git)
- Install ca-certificates (for HTTPS to api.anthropic.com)
- Install Claude Code CLI globally via npm
- Create non-root user `agent` at uid 1000

## Dockerfile skeleton

```dockerfile
FROM node:20-bookworm-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    ca-certificates \
  && rm -rf /var/lib/apt/lists/*

RUN npm install -g @anthropic-ai/claude-code

RUN useradd -m -u 1000 -s /bin/bash agent
RUN mkdir -p /workspace && chown agent:agent /workspace

USER agent
WORKDIR /workspace
```

No ENTRYPOINT or CMD — the full command is passed at `docker run` time by the daemon.

## Build command (for documentation/testing)

```bash
docker build -t outpost-agent:latest mcp/remote-worker/
```

## Notes

- No ENTRYPOINT means the container accepts arbitrary commands, which is correct — the daemon constructs the full `claude --print ...` invocation
- The `agent` user (uid 1000) matches the `--user 1000:1000` flag that WI-048 will pass at run time
- The `/workspace` directory must be pre-created and owned by `agent` so the bind mount works without root
- `--no-install-recommends` keeps image size minimal
- `rm -rf /var/lib/apt/lists/*` reduces image layer size
