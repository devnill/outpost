## Divergences to Fix

### 1. Remove poll_session

- Remove `poll_session` from the Component Map table (§1 MCP Servers — Tools column for session-spawner)
- Remove the entire `poll_session` Tool Definition section (§3)
- The implementation is synchronous; `spawn_session` blocks and returns the full result. There is no poll.

### 2. Fix spawn_session output schema (§3)

Current architecture says output is:
- `session_id`, `status: "running"`, `started_at`

Actual implementation returns (synchronously, after completion):
- `output`: session stdout
- `exit_code`: process exit code
- `session_id`: UUID
- `duration_ms`: execution time
- `error`: error message if failed
- `timed_out`: boolean (present when true)
- `token_usage`: object or null
- `output_truncated`: boolean (present when true)
- `full_output_path`: string (present when output was truncated)

Update the output schema to match. Keep the input schema as-is (it is correct).

### 3. Fix OUTPOST_CONCURRENCY → OUTPOST_MAX_CONCURRENCY (§8)

### 4. Fix spawn_remote_session worker_url → worker_name (§3)

### 5. Fix poll_remote_job worker_url → worker_name (§3)
