# Notes: WI-025 — Fix Integration Tests to Exercise MCP Tool Layer

## Context

`mcp/test_integration.py` Tests 2 and 3 call the remote-worker HTTP API directly using `aiohttp`:
- Test 2 (`test_get_job_newly_submitted_is_queued`): `http_session.get(f"{base_url}/jobs/{job_id}")`
- Test 3 (`test_delete_queued_job_cancels_it`): `http_session.delete(f"{base_url}/jobs/{job_id}")`

Both should instead call through the session-spawner MCP tool handlers, which is what users actually invoke. Test 4 (`test_reviewer_role_prefixes_prompt`) correctly uses `spawner_mod._handle_spawn_remote_session`.

## Fix

Read `mcp/test_integration.py` and the `worker_server` fixture to understand how `_remote_workers` is configured and the `http_session` fixture.

### Test 2 replacement

Replace the direct HTTP call with:
```python
result = await spawner_mod._handle_poll_remote_job({"job_id": job_id})
# _handle_poll_remote_job returns [TextContent(type="text", text="...")]
# Parse the JSON text to get the status
import json
data = json.loads(result[0].text)
assert data["status"] == "queued"
```

### Test 3 replacement

Replace the direct HTTP calls with:
```python
# Submit job first
# Then cancel via MCP handler
result = await spawner_mod._handle_cancel_remote_job({"job_id": job_id})
data = json.loads(result[0].text)
assert data["status"] == "cancelled"
# Verify poll also returns cancelled
poll_result = await spawner_mod._handle_poll_remote_job({"job_id": job_id})
poll_data = json.loads(poll_result[0].text)
assert poll_data["status"] == "cancelled"
```

## Notes

- `_handle_poll_remote_job` and `_handle_cancel_remote_job` both use `_get_http_session()` internally. The `http_session` fixture in `test_integration.py` sets `spawner_mod._http_session = session`. Verify this is still set correctly before calling the handlers.
- Both handlers return a list with a single `TextContent` item. Parse the text as JSON to get the actual response data.
- The `_remote_workers` global in spawner_mod must be set to point at the in-process worker (the `worker_server` fixture does this). Verify the fixture is properly injected into the test.
