# WI-049 Implementation Notes: Tests for containerized job execution

## File to modify

`mcp/remote-worker/test_server.py`

## Pattern

All tests patch `subprocess.Popen` or use `monkeypatch` to set `OUTPOST_AGENT_IMAGE`. The existing test file uses `pytest` and `pytest-asyncio`. Follow the same patterns as existing `_run_claude_job` tests.

## Test 1: Container mode activated when OUTPOST_AGENT_IMAGE is set

```python
def test_container_mode_uses_docker_run(monkeypatch, ...):
    monkeypatch.setenv("OUTPOST_AGENT_IMAGE", "outpost-agent:latest")
    # reload module-level constant or patch _agent_image
    with patch("subprocess.Popen") as mock_popen:
        mock_popen.return_value.__enter__.return_value = ...  # configure mock
        # call _run_claude_job with a test record
        ...
    call_args = mock_popen.call_args[0][0]  # first positional arg: the cmd list
    assert call_args[0] == "docker"
    assert "run" in call_args
```

Note: the module reads `_agent_image = os.environ.get(...)` at import time. Tests need to patch the module-level `_agent_image` variable directly (e.g., `monkeypatch.setattr(server_module, "_agent_image", "outpost-agent:latest")`) rather than setting the env var, since the module was already imported.
