# Notes: 012 — Fix Installation Paths

## Context

Three installation path bugs identified as critical findings CI1–CI3 in the cycle 2 review. All are mechanical fixes — no logic changes.

## Fixes

### 1. plugin.json args path

File: `.claude-plugin/plugin.json`

Current: `"args": ["${pluginPath}/mcp/server.py"]`
Fix: `"args": ["${pluginPath}/mcp/session-spawner/server.py"]`

The file `mcp/server.py` does not exist. The actual server is at `mcp/session-spawner/server.py`.

### 2. README.md pip install command

File: `README.md`, Installation section

Current: `pip install -r requirements.txt`
Problem: There is no root `requirements.txt`. Each component has its own.

Fix: Update to reference the session-spawner requirements file:
```
pip install -r mcp/session-spawner/requirements.txt
```

If the user also wants to run the remote worker, they would also need:
```
pip install -r mcp/remote-worker/requirements.txt
```

Either add both commands or update the docs to say "install requirements for the component you're using."

### 3. README.md claude mcp add command

File: `README.md`, MCP Server Setup section

Current: `claude mcp add outpost -- python /path/to/outpost/mcp/server.py`
Fix: `claude mcp add outpost -- python /path/to/outpost/mcp/session-spawner/server.py`

### 4. README.md tool list

File: `README.md`, Usage section

Current: lists `poll_session` as an available tool.
Fix: Remove `poll_session` from the tool list. The implementation is synchronous; `poll_session` does not exist.

## Verification

Check that `mcp/session-spawner/requirements.txt` and `mcp/remote-worker/requirements.txt` exist before updating the README.
